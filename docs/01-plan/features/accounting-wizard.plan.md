# 완전초보자를 위한 회계자료등록 마법사

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 정치자금 회계에 익숙하지 않은 초보 회계책임자가 계정/과목/지출유형 등 복잡한 코드 체계를 이해하지 못해 자료 등록에 어려움을 겪음 |
| **Solution** | 단계별 마법사(Wizard) UI로 자연어 질문에 답하면 시스템이 적합한 코드값을 자동 선택하여 등록을 완료하는 가이드 방식 |
| **Function UX Effect** | "어떤 종류의 거래인가요?" → "누구에게?" → "얼마?" 3단계 질문으로 복잡한 회계 입력을 단순화, 초보자도 1분 이내 등록 가능 |
| **Core Value** | 회계 전문지식 없이도 정확한 정치자금 장부 입력이 가능해져 선거 회계 진입 장벽을 대폭 낮춤 |

## 1. 배경 및 문제

### 현재 상황
- 수입내역관리/지출내역관리 페이지는 **계정 → 과목 → (지출유형 3단계) → 거래처 → 금액** 순서로 입력
- 계정(후보자등자산, 후원회기부금, 보조금, 보조금외지원금)과 과목(선거비용, 선거비용외) 개념을 모르면 입력 불가
- 지출유형은 11개 대분류 × 수십 개 중분류 × 소분류로 선택지가 너무 많음
- 초보 회계책임자는 "현수막 제작비는 어디에 입력해야 하나요?" 같은 기본 질문에서 막힘

### 대상 사용자
- 처음 선거 출마하는 후보자의 회계책임자
- 정당/후원회의 신규 담당자
- 회계 프로그램 사용 경험이 없는 자원봉사자

## 2. 목표

1. **초보자도 3단계 이내에 자료 등록 완료** (계정/과목/지출유형 직접 선택 없이)
2. **자연어 기반 거래 유형 분류** ("현수막 만들었어요" → 선거비용 > 거리게시용현수막)
3. **실수 방지**: 잘못된 코드 조합이 불가능하도록 마법사가 유효한 값만 제시
4. **기존 시스템과 100% 호환**: 마법사를 통해 등록한 데이터는 기존 수입/지출 페이지에서 동일하게 조회·수정 가능

## 3. 핵심 기능

### 3.0 사용자 여정 및 감정 설계

| Step | 사용자 행동 | 감정 목표 | 디자인 수단 |
|------|------------|----------|------------|
| 진입 | 사이드바에서 "간편등록 마법사" 클릭 | "이건 나 같은 초보자를 위한 거구나" | 메뉴명 "간편등록", 페이지 제목 하위에 안내 문구 |
| Step 1 | 카드 그리드 탐색 | "아, 이런 종류가 있구나. 그림으로 보니 쉽네" | 이모지 아이콘 + 짧은 설명, 검색 보조 |
| Step 1.5 | 기타 → 지출유형 선택 | "모르겠으면 이걸 고르라니, 다행이다" | 기본 추천값 + 도움말 문구 |
| Step 2 | 폼 입력 | "이것만 채우면 되는 거지? 간단하네" | 최소 필드, 기본값(오늘 날짜), 거래처 자동완성 |
| Step 3 | 자동 설정값 확인 | "시스템이 다 해줬네. 확인만 하면 되겠다" | 파란 배경의 자동설정 섹션 + 수정 가능 표시 |
| 성공 | 등록 완료 배너 | "됐다! 생각보다 쉬웠어" | 초록 성공 배너 + 등록 요약 + 다음 행동 버튼 |

**첫 방문 안내 문구** (Step 1 상단):
> "거래 종류를 선택하면 복잡한 계정·과목을 시스템이 자동으로 설정합니다. 카드를 골라보세요!"
>
> 폰트: 14px, 색상: text-muted (#6B7280), 한 번 본 후 로컬스토리지로 숨김 처리 가능

### 3.1 마법사 흐름 (지출)

```text
Step 1: 거래 유형 선택 (시각적 카드)
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│  🏢 사무소    │ │  📄 인쇄물    │ │  📢 광고/홍보  │
│  임대료,관리비 │ │  명함,공보물   │ │  신문,TV,인터넷 │
└──────────────┘ └──────────────┘ └──────────────┘
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│  🎽 소품      │ │  🚗 차량/이동  │ │  📱 전화/문자  │
│  어깨띠,모자   │ │  임차,유류비   │ │  통화,SMS발송  │
└──────────────┘ └──────────────┘ └──────────────┘
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│  👥 인건비    │ │  🏗 현수막/설치 │ │  📦 기타      │
│  사무원수당    │ │  현수막,무대   │ │  그 외 지출    │
└──────────────┘ └──────────────┘ └──────────────┘

→ 시스템이 자동으로: 계정 + 과목 + 지출유형1 + 지출유형2 설정

Step 2: 세부 정보 입력 (간소화된 폼)
- 거래처명 (+ 자동완성)
- 금액
- 날짜 (기본값: 오늘)
- 내역 설명 (자유 입력)
- 결제수단 (카드/현금/계좌이체)

Step 3: 확인 및 저장
- 자동 설정된 계정/과목/지출유형 표시 (수정 가능)
- "이 내용으로 등록하시겠습니까?" 확인
- 증빙서번호 자동 채번
- 증빙파일 첨부 (선택)
```

| 기타 | (기관별 첫 계정) | 선거비용 | 기타 | 사용자가 지출유형 직접 선택 |

### 3.4 "기타" 카드 선택 시 Step 1.5: 지출유형 직접 선택

"기타" 카드를 선택하면 바로 Step 2로 가지 않고, 지출유형 선택 화면(Step 1.5)을 거침:

```text
Step 1.5: 지출유형 선택 (기타 전용)
┌─────────────────────────────────────────────────────┐
│  어떤 종류의 지출인지 선택해주세요                       │
│                                                       │
│  지출유형1  [선거사무관계자 ▼]                           │
│  지출유형2  [사무원 ▼]          (선택 시 표시)           │
│  지출유형3  [-]                 (있을 때만 표시)          │
│                                                       │
│  💡 잘 모르겠다면 "선거사무소 > 기타"를 선택하세요        │
│                                                       │
│                            [← 이전] [다음 →]           │
└─────────────────────────────────────────────────────┘
```

- "기타" 외 카드는 지출유형이 자동 설정되므로 Step 1.5를 건너뜀
- Step 1.5에서 선택한 지출유형1로 과목(선거비용/선거비용외)이 자동 판별됨
- "잘 모르겠다면" 안내 문구로 초보자의 선택 불안 해소
- **진행 표시줄은 3단계 유지.** Step 1.5는 Step 1 영역 안에서 슬라이드 전환되는 하위 분기이며, 독립 단계가 아님. 사용자에게 "3단계"라는 약속을 깨지 않음

### 3.5 자연어 검색 보조

"기타" 카드를 선택하거나, 어떤 카드를 골라야 할지 모를 때:
- 텍스트 입력란에 "현수막", "사무실 전기세", "명함 인쇄" 등 키워드 입력
- 키워드 매핑으로 적합한 카드/지출유형 하이라이트 추천

### 3.6 정보 위계 (Information Hierarchy)

각 Step에서 사용자가 보는 순서:

| Step | Primary (가장 먼저) | Secondary | Tertiary |
|------|---------------------|-----------|----------|
| Step 1 | "어떤 종류의 거래인가요?" 질문 | 카드 그리드 | 검색바, 수입/지출 토글 |
| Step 1.5 | "어떤 종류의 지출인지 선택해주세요" | 지출유형 드롭다운 | 도움말 문구 |
| Step 2 | 선택한 카드 아이콘+이름 (맥락 유지) | 폼 필드 (거래처, 금액 순) | 증빙파일 첨부 |
| Step 3 | "등록 내용을 확인하세요" | 자동설정값 (파란 배경) | 입력 내용 요약 |

## 5. 기술 설계 방향

### UI 구조
```text
WizardPage
├── StepIndicator (1/3, 2/3, 3/3 진행 바)
├── Step1: TypeSelector (카드 그리드 + 키워드 검색)
├── Step2: DetailForm (거래처, 금액, 날짜, 내역, 결제수단)
├── Step3: ConfirmSave (자동설정값 표시 + 수정 가능 + 저장)
└── CustomerSearchDialog (거래처 검색)
```

### 상태 관리
- `useState`로 step(1~3), selectedType, formData 관리
- 이전 단계로 돌아가기(뒤로) 지원
- Step 3에서 계정/과목 수정 시 CodeSelect 재사용

### 데이터 흐름
```text
사용자 카드 선택 → lib/wizard-mappings.ts에서 코드 조회
  → useCodeValues()로 유효성 확인 (acc_rel 기반)
  → Step 2 폼 입력 → Step 3 확인
  → /api/acc-book insert + /api/evidence-file (선택)
```

## 7. 인터랙션 상태 명세

### 7.1 저장 성공 — 인라인 성공 배너 (confirm() 대체)

```text

│  ✓ 등록이 완료되었습니다!                              │

│  └──────────────────────────────────────────────┘     │
```

  - "등록은 완료되었으나 증빙파일 저장에 실패했습니다" + [재업로드] 버튼

- Step 3 영역을 성공 배너로 교체 (페이지 이동 없음)

- "추가 등록하기" → Step 1로 리셋 (Primary 버튼)
- "지출내역 보기" → /dashboard/expense 이동 (Outline 버튼)

## 8. 반응형 설계

### 8.1 뷰포트별 레이아웃

| 뷰포트 | 카드 그리드 | 폼 레이아웃 | 최대 너비 | 비고 |
|--------|-----------|-----------|----------|------|
| Desktop (≥768px) | 4열 | 2열 grid | 768px | 기본 경험 |
| Tablet (640-767px) | 3열 | 2열 grid | 100% - 48px | 사이드바 접힘 |
| Mobile (< 640px) | 2열 | 1열 stack | 100% - 32px | 터치 최적화 |

### 8.2 모바일 특화 조정

- **카드**: 최소 터치 타겟 44x44px 보장. 모바일에서 카드 높이 증가 (padding: 20px 16px)
- **Step indicator**: 모바일에서 라벨 텍스트 숨김, 숫자 dot만 표시
- **Step 2 폼**: 1열 스택, 각 필드 전체 너비. 금액 필드에 `inputmode="numeric"` 적용
- **Step 3 확인**: 자동설정 섹션과 입력내용 섹션 세로 스택
- **버튼**: 모바일에서 전체 너비 (`w-full`), 하단 고정 고려
- **검색바**: 모바일에서도 전체 너비, 상단 고정하지 않음 (스크롤과 함께 이동)

### 8.3 터치 인터랙션

- 카드 선택: `active` 상태에서 배경색 미세 변화 (탭 피드백)
- 스와이프로 Step 전환하지 않음 (폼 입력과 충돌 위험)
- 날짜 입력: 네이티브 date picker 사용 (`type="date"`)

## 9. 접근성 (Accessibility)

### 9.1 키보드 내비게이션

| 키 | 동작 |
|----|------|
| Tab | 카드 간 이동 (Step 1), 폼 필드 간 이동 (Step 2-3) |
| Enter / Space | 카드 선택, 버튼 클릭 |
| Escape | 거래처 검색 다이얼로그 닫기 |
| Arrow keys | 카드 그리드 내 방향 이동 (선택적) |

### 9.2 스크린리더 지원

- 카드: `role="radiogroup"` + 각 카드 `role="radio"` + `aria-checked`
- Step indicator: `aria-current="step"` + `aria-label="3단계 중 1단계: 유형선택"`
- 수입/지출 토글: `role="tablist"` + `role="tab"` + `aria-selected`
- 폼 필드: `<Label htmlFor>` 연결 (현재 구현됨)
- 에러 메시지: `aria-live="polite"` + `aria-describedby`로 필드 연결
- 성공 배너: `role="alert"` + `aria-live="assertive"`

### 9.3 색상 대비

- 본문 텍스트(#1A1A1A) on 배경(#F8F7F5): 대비율 15.2:1 (AAA 통과)
- 보조 텍스트(#6B7280) on 배경(#F8F7F5): 대비율 4.9:1 (AA 통과)
- Primary 버튼(#FFFFFF on #1B3A5C): 대비율 10.8:1 (AAA 통과)
- Accent 버튼(#FFFFFF on #D4883A): 대비율 3.1:1 (AA Large 통과, 14px Bold 이상에서 사용)

### 9.4 포커스 표시

- 모든 인터랙티브 요소에 `:focus-visible` 스타일: 2px solid var(--primary) + 2px offset
- 카드 포커스: 보더 색상 변경 + outline
