## 1. Overview

### 1.1 Purpose

현재 결산작업 페이지(`settlement/page.tsx`)와 수입지출보고서 페이지(`income-expense-report/page.tsx`)가 DB의 원시 데이터를 그대로 합산하고 있어, 선관위 정치자금 회계관리 프로그램의 보고서와 수치가 불일치합니다.

실제 DB(Fund_Data_1.db) 검증 결과 **데이터 자체는 정확**하지만, 선관위 프로그램이 적용하는 2가지 보정 로직이 프로젝트에 미구현되어 차이가 발생합니다.

### 1.2 Background — 검증된 차이 원인

#### 원인 1: 마이너스 수입 처리 (500,000원 차이)

ACC_BOOK에 "계좌입금오류반환처리"로 기록된 2건:
- `id=47`: 수입(INCM_SEC_CD=1), +500,000원
- `id=48`: 수입(INCM_SEC_CD=1), -500,000원

| 처리 방식 | 수입 반영 | 지출 반영 |
|-----------|----------|----------|
| **선관위 프로그램** | +500,000만 수입 | -500,000을 지출로 전환 |
| **프로젝트 현재** | +500,000 + (-500,000) = 상쇄 | 영향 없음 |

결과: 수입/지출 합계 각각 500,000원 차이

#### 원인 2: 계정별 지출 배분 차이 (1,870,675원 재배분)

선관위 보고서는 지출을 자금출처별로 재배분:

| 재배분 방향 | 금액 | 설명 |
|------------|------|------|
| 보조금 → 자산 | 1,866,665 | 보조금 4,415,000 중 2,548,335만 보전 인정, 잔액은 자산 부담 |
| 후원회기부금 → 자산 | 4,010 | 후원회기부금 잔액 4,010을 자산 부담으로 이동 |

결과: 자산 선거비용 지출이 99,325 → 1,970,000으로 증가

## 2. Scope

### 2.1 In Scope

- [ ] **마이너스 수입 보정 로직**: `acc_amt < 0`이고 `incm_sec_cd = 1`인 레코드를 지출로 전환 처리
- [ ] **수입지출보고서 보정**: 선관위 형식과 동일한 계정별 합산 결과 표시
- [ ] **결산작업 페이지 보정**: 계정/과목별 내역 테이블에 보정된 수치 반영

### 2.2 Out of Scope

- 선관위 자금출처별 충당 규칙(보전 재배분) — 현재 DB 데이터가 이미 계정별로 입력되어 있으므로, 마이너스 수입 보정만으로 충분한지 우선 검증 후 판단
- Excel 다운로드 로직 변경 (보정 로직 적용 후 자동 반영됨)
- 제출파일 생성 로직

## 3. Implementation Strategy

### 3.1 보정 유틸리티 함수 생성

`app/src/lib/accounting/adjust-negative-income.ts` 신규 생성:

```typescript
interface AccRecord {
  incm_sec_cd: number;  // 1=수입, 2=지출
  acc_sec_cd: number;   // 계정코드
  item_sec_cd: number;  // 과목코드
  acc_amt: number;      // 금액
}

/**
 * 마이너스 수입(acc_amt < 0, incm_sec_cd=1)을
 * 양수 지출(incm_sec_cd=2)로 전환
 */
function adjustNegativeIncome(records: AccRecord[]): AccRecord[]
```

## 4. Risk & Considerations

### 4.1 검증 필요 사항

- 마이너스 수입 보정만으로 선관위 보고서와 완전 일치하는지 확인
- 일치하지 않는 항목이 있다면 자금출처별 충당 규칙 추가 구현 검토

### 4.2 기존 기능 영향

- 결산 페이지의 잔액 = 재산(현금및예금) 검증 로직에 영향 없음 (잔액은 동일)
- 결산확정 저장(`opinion` 테이블) 시 보정된 수치로 저장

### 4.3 구현 복잡도

- **Low**: 유틸리티 함수 1개 + 기존 페이지 2곳 수정
- 예상 변경 파일: 3개 (신규 1 + 수정 2)

---

## 5. Success Criteria

1. 결산작업 페이지 수입/지출 합계가 선관위 보고서와 일치
2. 수입지출보고서 계정별 수치가 선관위 보고서와 일치
3. 기존 잔액 검증(잔액 = 현금및예금) 기능 정상 동작
4. 마이너스 수입이 없는 일반 데이터에서는 기존과 동일하게 동작
