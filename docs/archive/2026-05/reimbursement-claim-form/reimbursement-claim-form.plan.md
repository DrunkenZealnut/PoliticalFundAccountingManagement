## 1. 배경 및 필요성

### 1.1 선거비용 보전청구서란?

「공직선거법」 제122조의2·공직선거관리규칙 제51조의3에 따라 후보자(비례대표지방의원선거의 경우 추천정당)가 적법한 선거운동을 위해 지출한 보전대상 선거비용을 선거구 관할 선관위에 청구하는 공식 문서. 청구 기준 충족 시 선거구 부담으로 보전됨.

### 1.2 양식 종류

| 서식 | 대상 | 핵심 차이 |
|---|---|---|
| **서식 1** | 지역구지방의원, 지방자치단체의 장 | 선거사무소·연락소 단위, 자금원별(후보자자산/기부금/보조금/보조금외) 분류 |
| **서식 2** | 비례대표지방의원 | 정당 단위, 보조금 종류별(경상/선거/여청/장애인/청년) 분류 |

### 1.3 보전요건 요약

| 선거 종류 | 보전금액 | 조건 |
|---|---|---|
| 지역구지방의원·지자체장 | 100% | 당선·사망 또는 유효투표 15% 이상 |
| 〃 | 50% | 유효투표 10% 이상 ~ 15% 미만 |
| 비례대표지방의원 | 100% | 당선인이 있는 정당 |

### 1.4 청구 일정 (제9회 전국동시지방선거)

- **선거일**: 2026-06-03
- **청구기한**: **2026-06-15(월)** (선거일 후 12일)
- **추가청구 가능 기한**: 회계보고서 제출 기한(2026-07-03)까지
- **보전금 지급기한**: 2026-07-31(금)

### 1.5 현재 시스템 Gap

- `dashboard/reimbursement/page.tsx`: 보전 대상 체크 기능 + 부담비용 청구(서식7) 탭만 존재 → **서식 1·2 생성 기능 없음**
- `lib/excel-template/burden-cost-form.ts`: 서식 7 Excel 생성 구현 → **서식 1·2용 동등 모듈 없음**
- 자금원별(후보자자산/기부금/보조금) 자동 분류 로직 없음 — 현재 acc_book에 자금원 구분 컬럼은 존재하지만 보전청구 분류 매핑 함수 미구현
- 선거사무소·선거연락소 단위 분리 집계 로직 없음

## 2. 기능 요구사항

### 2.1 보전청구서 자동 생성 (FR-01)

- **입력**: 회계연도(orgId), 청구 단위(선거사무소/선거연락소), 양식 종류(서식1/서식2 자동 판별)
- **양식 자동 결정**: `organ.org_sec_cd` → 비례대표 정당이면 서식 2, 그 외 서식 1
- **출력**: 서식 1 또는 서식 2 형식의 Excel 파일

### 2.2 자금원별 자동 집계 (FR-02)

서식 1의 청구내역 표는 **자금원별** 5열 분류:

| 자금원 | 식별 기준 (acc_book.col_organ 또는 대응 코드) |
|---|---|
| **후보자자산** | 자기자금 / 차입금 |
| **후원회 기부금** | 후원회로부터의 수입 |
| **정당의 지원금 — 보조금** | 국고보조금 (경상/선거/여청 등) |
| **정당의 지원금 — 보조금외** | 정당 자체 자금 |
| **합계** | 위 4개 합산 |

→ DB의 수입거래 자금원(`col_organ` 등)을 매핑한 후, 각 지출 거래의 자금원을 추적해 합산하는 로직 필요. 재사용 가능 함수: `lib/accounting/balance-calc.ts`(있는 경우)

### 2.3 선거사무소·선거연락소 분리 집계 (FR-03)

- 선거연락소가 있는 경우 선거사무소 청구서는 연락소별 청구액을 **별도 행**으로 표기 후 합계 산정
- 선거연락소 청구서는 해당 연락소분만, 수령계좌란 미기재
- 데이터 소스: `organ` 또는 별도 `branch_organ` 구조 검토 필요 (현 schema 확인 후 결정)

### 2.4 보전청구 총액 산정 (FR-04)

- 선거비용과목(과목코드 = 선거비용)에 한정한 acc_book 지출 합계 (마이너스/취소 제외)
- 보전 대상 체크(`acc_print_ok = 'Y'` 또는 동등 플래그)된 행만 합산
- 통화 표기: 한글 "금 원(₩ )" 형식

### 2.5 청구인 정보 입력 (FR-05)

서식 1: 후보자, 선거사무장, 회계책임자 (3인 모두 기명·날인)
서식 2: 정당 대표자(시·도당 또는 정당선거사무소장), 선거사무장, 회계책임자

→ `organ` 테이블에 저장된 값을 기본값으로 채우되 사용자가 화면에서 수정 가능

### 2.6 수령계좌 정보 (FR-06)

- 정치자금 수입·지출 예금계좌 (후보자 명의)
- 비례대표는 신고된 중앙당/시·도당의 수입용 계좌
- 입력 필드: 예금주, 금융기관명, 계좌번호, 비고

### 2.7 부속서류 안내 체크리스트 (FR-07)

서식 1 첨부:
1. 정치자금 수입·지출부(선거비용과목) 사본 ✅ (기존 Excel export 활용)
2. 영수증 등 증빙서류 사본 ⚠️ (수기 첨부 안내)
3. 선거연락소별 보전청구서 사본 (해당 시)
4. 정치자금 수입·지출 통장 사본

화면에서 체크리스트로 표시하고, 1·3번은 시스템이 자동 생성 가능한 항목으로 별도 안내.

### 2.8 추가청구 모드 (FR-08)

- 청구기한(6/15) 후 누락분 추가청구 시 양식 제목을 "선거비용 보전청구서(추가)"로 표기
- 청구내역에 추가청구분 작성, 보전청구 총액은 기청구금액 포함 총액

### 2.9 보전 페이지 통합 (FR-09)

`dashboard/reimbursement/page.tsx`에 신규 탭 추가:

```
[ 선거비용 보전 (체크) ] [ 부담비용 청구 (서식7) ] [ 선거비용 보전청구서 (서식1·2) ]
```

세 번째 탭이 본 신규 기능. 첫 번째 탭의 체크 결과(`acc_print_ok`)를 입력으로 활용.
